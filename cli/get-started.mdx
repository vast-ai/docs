The `vastai` package bundles the CLI, Python SDK, and Serverless client in a single install.

## Install

Install the latest stable release from PyPI:

```text Text
pip install vastai
```

Requires Python 3.9+. View the package on [PyPI](https://pypi.org/project/vastai/) or browse the source on [GitHub](https://github.com/vast-ai/vast-cli).

## Quickstart

Get your API key at [https://cloud.vast.ai/cli/](https://cloud.vast.ai/cli/) and run the `vastai set api-key ...` command shown there. Don't share your api-keys. They authenticate every command from your account. To create restricted keys, see [create api-key](/cli/commands#create-api-key).

Run `vastai --help` for the full command list, or `vastai <command> --help` for a specific command's flags.

## Searching offers

The `search offers` command accepts the same filters and sort options as the web UI. For example, find single-GPU RTX 4090 offers:

```text Text
vastai search offers 'gpu_name=RTX_4090 num_gpus=1'
```

Run `vastai search offers --help` for the full list of fields and sort options.

## Launching instances

Create an instance with the ID of an offer returned from `search offers`. This creates an instance with the PyTorch image, 32 GB of disk, and SSH direct access:

```text Text
vastai create instance <offer_id> --image pytorch/pytorch --disk 32 --ssh --direct
```

You're billed for storage from creation until destroy, and for GPU while the instance is running.

To search and create in one step, use `vastai launch instance`. Run `vastai launch instance --help` for the full flag list.

## Adding an SSH key

Before you can SSH into an instance, add your public key to your Vast account:

```text Text
vastai create ssh-key "$(cat ~/.ssh/id_ed25519.pub)"
```

Replace the filename if your key lives elsewhere, or omit the argument to generate a new key pair. List registered keys with `vastai show ssh-keys`.

## Connecting over SSH

First boot can take a few minutes while the image pulls. Check status with:

```text Text
vastai show instances
```

Once the instance is running, get the SSH connection string:

```text Text
vastai ssh-url <instance_id>
```

Copy the printed `ssh` command and run it to connect. On bash or zsh, pipe the URL straight into ssh:

```text Text
ssh $(vastai ssh-url <instance_id>)
```

To view details for a specific instance, use `vastai show instance <instance_id>`. For container logs, use `vastai logs <instance_id>`.

## Starting and stopping

```text Text
vastai stop instance <instance_id>
```

```text Text
vastai start instance <instance_id>
```

You can stop an instance to avoid GPU charges, converting it into a storage unit. Storage is usually very cheap compared to GPU.
Starting an existing instance takes only a second or less, whereas creating a new instance can take much longer (to pull a large docker image), so maintaining a pool of stopped instances is useful for many applications.

To reboot an instance without stopping it, run:

```text Text
vastai reboot instance <instance_id>
```

You can [call stop/destroy instance from inside](/documentation/instances/docker-execution-environment) the instance using a special autogenerated instance apikey, to avoid exposing your main apikey.

## Copying data

Copy files between instances, between local and an instance, or between an instance and cloud storage:

```text Text
vastai copy <src_id>:/path/on/src <dst_id>:/path/on/dst
```

Run `vastai copy --help` for the full set of source and destination formats, or `vastai cloud copy --help` for cloud-storage transfers.

## Destroying instances

```text Text
vastai destroy instance <instance_id>
```

```text Text
vastai destroy instances <instance_id_1> <instance_id_2>
```

Destroy the instance when done to stop storage charges.

## Tips

Global flags work with any command:

- `--raw`: output machine-readable JSON, suitable for piping into `jq` or scripting
- `--explain`: show the underlying API call a command would make, without running it
- `--curl`: print an equivalent `curl` invocation
