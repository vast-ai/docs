Async/await client for Vast Serverless. Identical to [`Serverless`](/sdk/python/serverless/serverless) in configuration, but `queue_endpoint_request` returns a `Coroutine` that resolves to the response dict when awaited, instead of a `ServerlessRequest` future.

Typically obtained from an `AsyncClient` via `AsyncClient.serverless()` so it can share the client's `aiohttp` session and SSL context.

## Import

```python
from vastai import CoroutineServerless
```

Or, via an `AsyncClient`:

```python
from vastai import AsyncClient

async with AsyncClient(api_key="...") as client:
    sl = client.serverless()
```

## Methods

### queue_endpoint_request

```python
queue_endpoint_request(
    endpoint: Endpoint,
    worker_route: str,
    worker_payload: dict,
    session: Optional[Session] = None,
    serverless_request: Optional[Union[ServerlessRequest, RequestStatus]] = None,
    cost: int = 100,
    timeout: Optional[float] = None,
    worker_timeout: Optional[float] = 600,
    retry: bool = True,
    max_retries: Optional[int] = None,
    stream: bool = False,
) -> Coroutine[Any, Any, dict]
```

Return a coroutine that resolves to the response dict from the worker when awaited. Pass a `ServerlessRequest` or `RequestStatus` in `serverless_request` to observe request lifecycle state externally.

All other configuration and lifecycle methods (`get_endpoint`, `close`, `is_open`, `get_ssl_context`) are inherited from the base — see [`Serverless`](/sdk/python/serverless/serverless) for the full surface.

## Example

```python
import asyncio
from vastai import AsyncClient

async def main():
    async with AsyncClient(api_key="YOUR_API_KEY") as client:
        sl = client.serverless()
        endpoint = await sl.get_endpoint(name="my-endpoint")
        coro = sl.queue_endpoint_request(
            endpoint=endpoint,
            worker_route="/v1/completions",
            worker_payload={"prompt": "Hello"},
            cost=100,
        )
        result = await coro
        print(result)

asyncio.run(main())
```
